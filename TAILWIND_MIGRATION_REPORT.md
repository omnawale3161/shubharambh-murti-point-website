# Tailwind CSS v4 Migration Report

## Summary

The project was migrated from Tailwind CSS v3 to the latest stable Tailwind CSS v4 release without redesigning the UI or changing business logic.

## Version Inventory

- Previous Tailwind CSS: `3.4.19`
- New Tailwind CSS: `4.3.0`
- Next.js: `16.2.9`
- PostCSS: `8.5.15`
- Previous PostCSS plugins: `tailwindcss`, `autoprefixer`
- New PostCSS plugin: `@tailwindcss/postcss`

## Previous Configuration

- Configuration file: `tailwind.config.ts`
- Content sources: `app`, `components`, and `lib`
- Custom colors:
  - `maroon`: `#780B22`
  - `gold`: `#B8872E`
  - `ivory`: `#FFFDF7`
  - `beige`: `#EFE5D2`
  - `ink`: `#241A16`
- Custom shadow: `shadow-premium`
- Fonts: system font stack defined in `app/globals.css`
- Animations: custom CSS `riseIn`, `zoomIn`, and reduced-motion rules
- Plugins: none
- Dark mode: no custom dark-mode setup

## Migration Changes

- Ran the official `@tailwindcss/upgrade@4.3.0` process.
- Replaced the v3 Tailwind directives with `@import "tailwindcss";`.
- Migrated custom colors and `shadow-premium` into the CSS-first `@theme` block.
- Removed the legacy `tailwind.config.ts` file.
- Replaced the PostCSS `tailwindcss` plugin with `@tailwindcss/postcss`.
- Removed the now-unnecessary `autoprefixer` dependency.
- Added Tailwind v4 border-color compatibility styles generated by the official upgrader.
- Updated renamed v4 utility semantics:
  - v3 `shadow` became `shadow-sm`
  - v3 `shadow-sm` became `shadow-xs`
  - `rounded-[2rem]` became the equivalent `rounded-4xl`
- Preserved custom CSS fonts, animations, accessibility rules, responsive behavior, brand colors, and opacity variants.
- Made the existing Playwright verification port configurable to avoid collisions with unrelated local servers.
- Scoped one existing E2E assertion to the main content region for deterministic verification.
